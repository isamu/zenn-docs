## この記事でわかること

- **MulmoClaude** ってそもそも何？（ざっくりでOK）
- 目玉機能 **コレクション**＝会話だけで作れる“自分専用アプリ”
- 実例3つ（カメラのレンズ📷 / 映画🎬 / レストラン🍽）と、**今すぐ試す**手順

「専用アプリを入れるほどじゃないけど、メモだと探せない」——そんなデータ、ありませんか。
そこにちょうどハマったのが MulmoClaude のコレクションでした。

## MulmoClaude とは？（30秒で）

**話しかけると、結果が“見える”アプリ**です。ふつうの AI チャットと違い、答えが文章だけでなく
**資料・表・画像・スライド、そして“アプリ”** になって画面に出てきます。作ったものは全部
**手元のただのファイル**として残るので、データはあなたの手元に。

くわしい紹介とインストールは上のリンクへ。ここからは目玉機能の **コレクション** に絞って説明します。

## コレクション＝会話で作る「自分専用アプリ」

コレクションは MulmoClaude の一番の目玉です。

ポイントは、**小さな JSON ファイル1枚（`schema.json`）に「どんなデータを持つか」を書くと、
それだけでアプリになる**こと。データベースも、画面のコーディングもいりません。

しかも **自分で JSON を書く必要はありません**。チャットでお願いするだけで Claude が用意します。

> 「ウイスキーを記録するコレクション作って。銘柄・産地・度数・評価で。」

これだけで、

- 入力フォーム
- 一覧の **表**
- 付箋を列でドラッグする **カンバン**
- **カレンダー** / **ダッシュボード**

が自動で出てきます。あとから「優先度フィールド足して」「これはカンバンで見たい」と言えば
**その場で作り替わる**。コードを書く代わりに“しゃべって作る”ので、これを
**vibe crafting（バイブ・クラフティング）** と呼んでいます。

## カスタムビュー＝“見え方”は言葉で頼める

標準の表で物足りないとき——「カレンダーで」「棒グラフで」「図鑑みたいに並べて」——と頼むと、
**Claude が専用の表示画面（カスタムビュー）を作ってくれます**。データが増えれば自動で反映。
次の3つの実例は、まさにこの“専用の見え方”がポイントです。

## 具体例：趣味を“自分専用アプリ”にする

### 📷 カメラのレンズ → 「図鑑」ビュー

レンズ沼にハマると「今 何 mm をカバーできてる？」「F2.8 通しあったっけ？」「これ売ったんだっけ？」
が分からなくなります。そこで 1 本 = 1 レコードでスペックを記録。

専用の **「図鑑」ビュー** がこれ:

- 焦点距離を **横バー** で表示（広角〜望遠がひと目。単焦点はドット）
- **マウントごとに色分け**、売ったレンズは薄く斜線で“過去のもの”が分かる
- **「35mm換算」トグル** で、APS-C もマイクロフォーサーズも同じ土俵に揃えて画角を比較
- 結果、「85mm の画角、持ってる？」が **見れば分かる**

![レンズの「図鑑」ビュー。焦点距離が横バーで並び、35mm換算トグルでフォーマット横断で画角を比較できる](/images/mulmocloud/collections/lens.png)

### 🎬 映画リスト → 「シネマ」ビュー

観たい映画と観た映画を、ジャンル・監督・視聴方法・評価・鑑賞日で記録。
**観た作品だけ ★ と鑑賞日の欄が出る**ので入力もスッキリ。

専用の **「シネマ」ビュー** は **Netflix 風**。ポスターが棚で並び、ジャンル別の横スクロール＋検索つき。
「未鑑賞の SF で評価高いやつ」みたいな絞り込みもチャットでお願いするだけ。

![映画の「シネマ」ビュー。Netflix 風にポスターがジャンル別の棚で横スクロール表示される](/images/mulmocloud/collections/cinema.jpg)

### 🍽 レストラン → 標準ビューだけでも十分

一番シンプルな例。カスタムビューを作らず、素の表のままでも便利です。

- 行きたいお店をメモして、行ったら「訪問済み」にチェック
- **訪問済みにすると評価欄がにゅっと出てくる**（行く前に星はつけられない、を形にしたもの）
- ジャンル・地域で絞れるので「今度の出張、近場で前に行きたかった店あったかな」がすぐ引ける

---

3つに共通する“型”はこれ:

1. **行きたい / 欲しい（wishlist）→ 体験した（記録）** の流れ
2. **状態**（持ってる / 売った・観た / 未見・行った / 未訪問）で分けられる
3. **そのジャンルだけの見え方**（焦点距離バー、ポスター棚…）

だから「趣味」ならだいたい何でも乗ります。読んだ本、行ったライブ、育てている植物、ボドゲ、コーヒー豆……。

## 今すぐ使う（2通り）

まだ起動していない人は、ターミナルで **`npx mulmoclaude@latest`**。開いたら次のどちらかで始められます。

### A. 自分でゼロから作る（30秒）

チャットにこう打つだけ:

> 「映画リストのコレクションを作って。タイトル・ジャンル・監督・評価・観たかどうかで。」

Claude が `schema.json` を用意して、`/collections` に表と入力フォームが現れます。
あとは「インセプション追加して、★5」のように足していくだけ。
「ポスターを棚で見たい」と言えばカスタムビューも付きます。

### B. 公開済みのコレクションを取り込む（ワンクリック）

人が作ったコレクションを、そのまま自分の環境で使えます。

1. `/collections` を開く →「**発見**」タブ
2. 一覧から選んで **インストール**
3. すぐ使える（中身は自由に差し替え）

今回作った **カメラのレンズ（図鑑ビュー付き）** もここに公開したので、カメラ好きはそのままどうぞ。

## まとめ

- **MulmoClaude** = 話すと結果が“見える”アプリ（Claude Code の GUI フロント）
- **コレクション** = `schema.json` 1枚で動く“自分専用アプリ”。**会話で作れて、その場で作り替えられる**
- **カスタムビュー** = 「こう見たい」を言葉で頼める
- 趣味の管理はこの相性が抜群。まずは **「◯◯リスト作って」** から始めてみてください。
