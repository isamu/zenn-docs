[MulmoTerminal](https://github.com/receptron/mulmoterminal) は、[Claude Code](https://claude.com/claude-code) をブラウザのターミナルで動かすツールです。`npx mulmoterminal@latest` 一発で、サーバ上の本物の PTY で `claude` が起動し、ブラウザの [xterm.js](https://xtermjs.org/) に流れてきます。

この記事では、その中でも個人的にいちばん"効く"**グリッドビュー**にフォーカスして、推し機能を5つ紹介します。ひとことで言うと **「複数の Claude Code エージェントを1画面のダッシュボードで束ねて並列運用する」** ための画面です。

```sh
npx mulmoterminal@latest
```

起動したらツールバーの **Grid（🔲）** アイコンでグリッドビューに切り替えます。

![グリッドビューの全景](/images/mulmoterminal-grid/01-overview.png)

## 1. 最大9分割 → タブで81枚、複数エージェントを一望

グリッドは1本のフラットなセルの並びで、**セル数に応じてレイアウトが自動で育ちます**（1 → 2 → 2×2 → 3×2 → 4×2 → 3×3）。9枚を超えると**タブ（ページ）**にあふれ、最大 **81 セッション**まで扱えます。

セルを閉じるとリストが自動で詰め直され、後ろのセルがページ境界を越えて前へ流れてきます。「AI エージェントを何体も同時に走らせて、1画面で見張る」体験がこれで成立します。

## 2. 各セルが独立した"本物"の Claude セッション

各セルはサーバ上の**本物の PTY で動く `claude`** です（`-p`/headless ではなく、対話モードの CLI をそのまま TTY 経由で中継）。空セルのランチャーからは、

- ディレクトリを選んで**新規起動**
- そのディレクトリの**既存セッションを resume**
- `script.json` の**コマンドを実行**（dev server やテストをセルとして常駐）

を選べます。

![セルのランチャー](/images/mulmoterminal-grid/02-launcher.png)

しかも**表示中のページ以外もサーバ側で走り続け**、ページを戻すと再アタッチします。うっかりタブを閉じそうになると確認ダイアログ（0.6.0 の新機能）が出て、動いているセッションを守ってくれます。

:::message
**コストの話。** ここで動くのは API でも headless でもなく、いつも使う**対話モードの `claude` そのもの**です。だから使用量は **あなたの Claude プラン（Max など）の枠内**でカウントされ、トークン従量の API 課金は発生しません。**グリッドで何体並列に回しても、料金は"いつものサブスク"の中**（プランの利用上限の範囲）で収まります。
:::

## 3. git worktree 分離 — 同じリポジトリを複数エージェントで安全に並行作業

これがグリッドビューの**いちばんの推し**です。

対象ディレクトリが git リポジトリなら、セルを**使い捨ての worktree**（作業ツリーは別・`.git` は共有）で起動できます。つまり **複数エージェントが同じリポを衝突なしで同時にいじれる**。「3体のエージェントで3つの機能を並行実装 → 3つの PR」が現実的になります。

さらに、セル内で開発ループが完結します。

- ヘッダーに **ahead / dirty バッジ**（ベースブランチとの差分）
- **diff パネル**で変更ファイル＋パッチを閲覧
- **Commit**（Claude 自身に良いメッセージでコミットさせる）→ **Push** → **Open PR** までワンクリック

![worktree セルの diff と PR 操作](/images/mulmoterminal-grid/03-worktree.png)

## 4. ズーム＆フィルムストリップ

1つのセルを **⤢ でズーム**すると、そのセルが大画面を占有し、残りは下部に**横スクロールのライブ帯（フィルムストリップ）**として並びます。ズーム中も全セルがライブのまま（Vue の Teleport で大画面へ差し込んでいるので、接続は切れません）。

「今はこれに集中、でも他のエージェントも視界に入れておく」を両立できます。

![ズームとフィルムストリップ](/images/mulmoterminal-grid/04-zoom.png)

## 5. Manual / Auto の並べ替え＋ステータス可視化で"呼び戻される"

ツールバーのトグルで並び順を切り替えられます。

- **Manual**：各セルの **◀▶** で手動並べ替え
- **Auto**：**要対応（waiting）→ アイドル → 動作中（working）** の安定ソートで、**あなたの操作待ちのセルが自動で前に浮く**

加えて、セルは**枠の色とドット**で状態を表示（青＝作業中／琥珀＝要対応）。そして **作業が終わって応答待ち（要対応）になると、音で知らせて**くれます（アテンション音）。**動的 favicon**（タブのアイコンが状態で変わる）も合わせると、**別のことをしていても「終わったよ／手を貸して」と向こうから呼び戻してくれます**。放置監視にぴったりです。

![各セルヘッダの ◀▶ とツールバーの並べ替えトグル](/images/mulmoterminal-grid/05-order.png)

## ＋α：`.mulmoterminal.json` でセルごとに色・名前・音を変える

グリッドにセルが増えてくると「どれがどのプロジェクト？」となりがちです。そこで、**各プロジェクトのディレクトリに `.mulmoterminal.json` を置く**と、そのディレクトリで開いたターミナル（＝そのセル）だけ、**バッジ名・色・テーマ・端末パレット・通知音**を個別にカスタムできます（アプリ全体のテーマはそのまま）。

![ディレクトリごとに背景色とバッジを変えたグリッド](/images/mulmoterminal-grid/06-dircolors.png)

たとえば本番リポは 🔥 赤、ステージングは 🧪 シアン、実験は 🚀 紫…と染め分ければ、ひと目でどのセルが何か分かります。設定ファイルはこれだけ:

```jsonc
{
  "name": "PROD · payments",            // ヘッダーに出るバッジ名
  "badgeColor": "#cf222e",              // バッジ色（#rrggbb）
  "theme": "nord",                      // 端末パレット: midnight | nord | daylight | solarized
  "colors": { "background": "#190a23", "cursor": "#ff2e63" }, // 個別の上書き
  "sound": "./.mulmoterminal/alert.mp3" // このディレクトリの通知音（相対パス）
}
```

全項目オプション。本番リポは赤バッジ＋専用の警告音、実験用は別の色…と分けておけば、**セルを見た瞬間にどのプロジェクトか分かり、音でも聞き分けられます**（機能5のアテンション音もディレクトリ単位で変えられます）。変更は次にそのターミナルを開いたときに反映されます。

## おまけ

- **二重オープン警告**：別セルで開いているセッションを resume しようとすると確認が出ます（取り違え防止）。
- **GitHub クイックリンク**：セルのリポジトリの Repo / Issues / PRs をヘッダーから直接開けます。

## まとめ

MulmoTerminal のグリッドビューは、

1. 自動レイアウト＋タブで**最大81セッションを一望**
2. 各セルが**本物の Claude セッション**（裏でも走り続ける）
3. **worktree 分離**で1リポジトリを安全に並行作業＋セル内 PR
4. **ズーム＆フィルムストリップ**で集中と俯瞰を両立
5. **Auto 並べ替え＋状態可視化**で放置監視できる

という構成で、**「AI エージェントを何体も同時に回す」** ための運用画面になっています。しかも動かしているのは本物の対話 `claude`。だから **何体並列に回しても料金はいつもの Claude サブスクの枠内**（プラン利用の範囲）で、終わったら音で呼び戻してくれます。

```sh
npx mulmoterminal@latest   # http://localhost:34567 が開きます
```
